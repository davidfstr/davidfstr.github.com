Disqus Reverts IE 8+ to Compatibility Mode

MISSING_DATE – Filed under:

This is a test article that includes Disqus commenting.

In IE 8+ this webpage will revert to Compatibility Mode instead of Standards Mode when Disqus loads unless the following is put in the [head] tag: 

      [!-- IE 8+: Hide the Compatibility View button. --]
      [meta http-equiv="X-UA-Compatible" content="IE=edge"]

Unfortunately the X-UA-Compatible does not pass the W3C validator, so using that header breaks validation for a site.

So in summary there are two problems:

  1. Disqus generates weird HTML that puts IE 8+ into Compatibility Mode.
  2. X-UA-Compatible is not recognized as a valid [meta] HTTP header.
    • I have filed bugs to get this fixed, but I no longer have the energy to keep sheparding these bugs, especially since I have eliminated the use of Disqus on my blog, which obviates the need to do any of this.
    • It would probably be best for someone on the IE team to push the X-UA-Compatible header (or a new UA-Compatible variant) towards standardization.
